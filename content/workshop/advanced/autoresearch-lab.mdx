<Timer minutes={20} />

## Step 1: Gemini에 저장소 연결

<Steps>

### Gemini 웹 접속

[gemini.google.com](https://gemini.google.com)에 접속합니다.

### + 버튼 → 코드 가져오기

입력창 왼쪽의 **+** 버튼을 누르고, 메뉴에서 **코드 가져오기**를 선택합니다.

![Gemini 도구 메뉴 — 코드 가져오기 선택](/workshop/advanced/gemini-tools-menu.png)

### 저장소 URL 입력 → 가져오기

"GitHub 저장소 또는 브랜치 URL" 입력란에 아래 URL을 붙여넣고 **가져오기**를 클릭합니다.

```
https://github.com/karpathy/autoresearch
```

![코드 가져오기 — autoresearch 저장소 URL 입력](/workshop/advanced/gemini-import-code.png)

### 첫 질문 입력

저장소가 연결되면, 아래 질문을 입력합니다:

```
이 저장소의 program.md와 train.py, prepare.py를 읽고,
이 프로젝트가 어떻게 작동하는지 한국어로 설명해줘.
특히:
- 사람이 하는 일과 AI가 하는 일을 구분해서
- 실험 루프가 어떻게 반복되는지
- "NEVER STOP"이라는 지시가 왜 필요한지
```

</Steps>

**관찰 포인트:**
- `program.md`가 AI의 행동을 지시하는 **유일한 파일**이라는 것
- 평가 기준(`val_bpb`)과 판단 기준(단순성, keep/discard)이 어떻게 설계되어 있는지
- "사람은 코드를 안 건드린다. 지시서만 다듬는다"는 핵심

## Step 2: Gemini Canvas로 인포그래픽 만들기

> "어려운 걸 쉽게 설명할 수 있을 때, 진짜 이해한 겁니다."

<Callout type="warning">
**가장 중요한 목표: "내가 이해하기 쉬운 형태"로 만드는 것**

예시 프롬프트를 그대로 써도 되지만, 핵심은 <strong>본인이 autoresearch를 이해한 방식</strong>으로 만드는 것입니다.
같은 개념이라도 사람마다 "아, 이거구나!" 하는 순간이 다릅니다.
자기만의 비유, 자기만의 구조로 만들어보세요.
</Callout>

### Canvas 모드 켜기

Gemini에서 도구 버튼을 누르고 **Canvas**를 선택해야 Canvas 모드가 활성화됩니다.

![Gemini 도구 메뉴 — Canvas 선택](/workshop/advanced/gemini-canvas-mode.png)

### 인포그래픽 주제 예시

아래 중 하나를 골라도 좋고, 자유롭게 만들어도 좋습니다:

- <strong>"사람 vs AI — 누가 뭘 하나?"</strong> — 사람은 program.md 한 장만 쓰고, AI가 나머지 전부(코드 수정, 실험, 평가, 기록)를 한다는 역할 분담을 한 장으로
- <strong>"자고 일어나면 100번"</strong> — 5분 x 12번/시간 x 8시간 = 약 100번. 사람이 자는 동안 AI가 쌓아놓는 실험 결과를 타임라인으로
- <strong>"파일 3개로 돌아가는 연구소"</strong> — prepare.py(고정), train.py(AI가 수정), program.md(사람이 다듬음). 이 구조가 왜 강력한지를 다이어그램으로

### 예시 프롬프트

아래 프롬프트를 참고용으로 제공합니다. 그대로 써도 좋지만, 본인만의 관점으로 수정하면 더 좋습니다.
이 프롬프트로 만든 [결과물 예시](https://gemini.google.com/share/540262a71570)도 참고해보세요.

<Tabs items={['예시 프롬프트']}>
<Tabs.Tab>

```
Gemini Canvas로 인터랙티브 웹페이지를 만들어줘.
Andrej Karpathy의 autoresearch 프로젝트를 초등학생도 이해할 수 있게 설명하는
한 장짜리 인포그래픽 페이지야.

구성 (위에서 아래로 스크롤):

1. 제목 영역
   - 제목: "autoresearch — AI가 밤새 실험하는 법"
   - 부제: "사람은 지시서 한 장만 쓴다. 나머지는 AI가 알아서."

2. 파일 3개 영역
   - 카드 3개를 가로로 배치
   - prepare.py: 🔒 자물쇠 아이콘, "고정 — 아무도 안 건드림", 회색 배경
   - train.py: 🤖 로봇 아이콘, "AI가 수정", 파란 배경
   - program.md: 👤 사람 아이콘, "사람이 다듬음", 초록 배경

3. 실험 루프 영역
   - 원형 다이어그램으로 루프 표현:
     코드 수정 → 5분 실험 → 점수 측정 → 좋으면 유지 / 나쁘면 버림 → 다시 시작
   - 가운데에 "이 루프를 밤새 ~100번 반복"

4. Before/After 영역
   - 왼쪽: 😴 "밤 11시 — 사람이 program.md 쓰고 잠든다"
   - 오른쪽: ☀️ "아침 7시 — 100번의 실험 결과가 쌓여있다"

5. 한 줄 요약
   - "코드를 프로그래밍하는 시대에서, 지시서를 프로그래밍하는 시대로."

디자인:
- 한국어로 작성
- 다크 모드 배경에 밝은 카드
- 섹션 사이에 충분한 여백
- 각 영역에 이모지 아이콘 활용
```

</Tabs.Tab>
</Tabs>

## Step 3: 옆 사람과 공유 + 대화

2인 1조로 진행합니다. 아래 순서대로 따라가세요.

### 1라운드: 인포그래픽 교환 (4분)

서로 Canvas 링크를 공유하고, **상대방의 인포그래픽을 1분간 혼자 읽습니다** (설명 없이).
그 다음 아래 두 문장을 각각 하나씩 말해줍니다:

- **"이 부분이 바로 이해됐다"** — 어떤 영역이 가장 직관적이었는지
- **"이 부분은 한 번 더 봐야 했다"** — 어디서 멈칫했는지

> 왜 설명 없이 먼저 읽나요? — 설명이 필요 없이 이해되는 인포그래픽이 좋은 인포그래픽입니다.
> "한 번 더 봐야 했다"는 곳이 곧 개선 포인트입니다.

### 2라운드: "내 업무라면?" 대화 (6분)

서로에게 이 질문을 던져보세요:

> **"autoresearch에서는 AI가 밤새 코드를 개선하잖아. 우리 업무에서도 이런 게 가능하다면, 뭘 맡기고 싶어?"**

대화 예시:
- "코드 리뷰 체크리스트를 AI가 계속 다듬어주면 좋겠는데"
- "기획서 템플릿을 과거 피드백 보고 알아서 개선해주면 어떨까"
- "테스트 시나리오에서 매번 빠지는 걸 AI가 찾아서 추가해주면 좋겠다"

<Callout type="info">
지금 나온 답변이 Phase 2에서 직접 만들 것입니다.

`program.md` = 코드 리뷰 지침서 / 기획서 리뷰 기준 / TC 설계 규칙
— 다음 세션에서 이걸 실제로 만들고, AI가 자동으로 개선하게 시킵니다.
</Callout>
