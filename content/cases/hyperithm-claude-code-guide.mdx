## 왜 이 사례가 중요한가

대부분의 팀이 AI 코딩 도구를 "각자 알아서" 사용합니다.
하이퍼리즘은 **사내 가이드를 작성해 팀 전체의 AI 활용을 표준화**하고, 이를 기술 블로그에 공개했습니다.

암호화폐 금융이라는 보안이 중요한 도메인에서, Claude Code를 어떻게 안전하고 효율적으로 사용하는지 구체적인 설정과 팁을 공유한 점이 실용적입니다.

---

## 가이드 핵심 내용

### Claude Code 기본 설정

하이퍼리즘 IA(Intelligent Automation) 팀이 개발자 생산성 향상을 위해 정리한 가이드로, 다음 내용을 다룹니다:

- **설치 및 초기 설정** — 터미널 기반 에이전트의 시작
- **권한 관리** — 셸 명령 실행, 파일 수정, 외부 API 접근 시 사용자 승인 프로세스
- **MCP 서버 설정** — 외부 도구와의 연동 구성
- **메모리 관리** — User Memory(개인 설정)와 Project Memory(Git 관리)의 분리 운영

### CLAUDE.md 활용 전략

Claude Code는 현재 작업 디렉토리에서 루트 디렉토리까지 재귀적으로 `CLAUDE.md` 또는 `CLAUDE.local.md` 파일을 탐색합니다.

하이퍼리즘은 이를 활용해:
- **프로젝트별 코딩 컨벤션**을 CLAUDE.md에 명시
- **개인 설정**은 CLAUDE.local.md로 분리 (Git에서 제외)
- **디렉토리별 특화 규칙**을 계층적으로 관리

### MCP 서버 구성

외부 서비스와의 연동을 위한 MCP(Model Context Protocol) 서버 설정 방법을 포함합니다.

## 핵심 인사이트

### 1. 사내 가이드가 팀 생산성의 출발점이다

개인이 AI를 잘 쓰는 것과 팀 전체가 일관되게 잘 쓰는 것은 다릅니다.
가이드 하나가 팀의 AI 활용 수준을 균일하게 올립니다.

### 2. 보안과 편의의 균형

금융 도메인에서 AI 코딩 도구를 도입할 때, 권한 관리와 메모리 분리는 필수입니다.
하이퍼리즘의 접근은 보안을 포기하지 않으면서도 생산성을 확보하는 현실적인 모델입니다.

### 3. 가이드를 공개하면 커뮤니티가 함께 발전한다

사내 가이드를 외부에 공개함으로써 다른 팀들도 참고할 수 있게 했습니다.
이런 공유 문화가 한국 개발 생태계의 AI 역량을 함께 높입니다.
